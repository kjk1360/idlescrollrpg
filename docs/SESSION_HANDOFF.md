# Session Handoff

This file is the first document a future Codex session should read before continuing work.

## User Goal

Build a Rust-first belt-scroll Idle RPG production platform, not just a single prototype game.

The target game style is:

- 2D belt-scroll RPG like Dungeon Fighter-style movement
- pseudo-3D movement through an `x` progress axis and a lane/depth axis
- endless movement to the left
- player units and enemy units fight automatically
- formation, unit groups, waves, maps, skills, equipment, drops, and behavior patterns are data-driven

The target tool style is:

- genre-specific toolchain, closer to RPG Maker than Unity/Unreal/Godot
- sheet-like Data Studio for table/field/row editing
- explicit Code Generate button/API
- generated Rust data types before runtime
- runtime only loads data into generated types and builds relation caches
- Codex should be able to use local CLI/API to validate, edit, generate, and build data
- UI tooling should feel closer to web UI authoring than Unity UGUI

## Key Design Agreement

- Build the first game and the dedicated toolchain together.
- Do not build a broad generic engine first.
- Data Studio schema is the source of truth for data types.
- Schema changes require explicit Code Generate.
- Row value changes do not require Code Generate; they require Data Build.
- Runtime must not create classes dynamically.
- Runtime uses generated Rust types plus loaded/cached data instances.

## Current State

Implemented:

- Rust workspace
- `belt_core`: deterministic belt-scroll auto-battle simulation
- `data_studio_core`: schema, row data, relation field kinds, validation, fingerprint status
- `belt_tools`: CLI for simulation, validation, status, codegen, data build
- `projects/sample`: file-based sample data project
- `crates/generated_data`: generated Rust crate from the sample schema
- `simulate --project projects\sample --map endless_left_road`: file-data-driven battle simulation

Important commands:

```powershell
cargo test
cargo run -p belt_tools -- simulate
cargo run -p belt_tools -- simulate --project projects\sample --map endless_left_road
cargo run -p belt_tools -- data-status --project projects\sample
cargo run -p belt_tools -- validate --project projects\sample
cargo run -p belt_tools -- codegen --project projects\sample --out crates\generated_data\src
cargo run -p belt_tools -- data-build --project projects\sample --out build\sample_data
```

Expected current sample status after codegen/data-build:
